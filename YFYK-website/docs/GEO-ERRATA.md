# V3.0 技术文档勘误

## 已确认修正

- 文件名为 `V3.0`，正文首页仍写 `v2.0.0 | 2026-05-8`。正式版本号待项目负责人确认。
- 删除“引用准确率提升 300%”等无可核验证据的示例，不进入正式页面和 Schema。
- Core Web Vitals 使用 INP `< 200ms`，不再使用已替代的 FID 指标。
- 中国站点默认部署改为国内云 + CDN/WAF。Vercel 仅作为可选海外或预览环境。
- C2PA 未接入签名链前，只能标记为 `pending`。
- 百度反向 DNS 校验放在 CDN/WAF 或缓存层，不在 Edge 中间件阻塞主请求。

## 工程解释

- Next.js App Router 不能在同一目录同时放置 `page.tsx` 和 `route.ts`。页面 Markdown 使用内部动态 API 路由，并通过中间件提供公开 `.md` 地址。
- `llms.txt` 是站点摘要与发现信号，不替代 `robots.txt`、sitemap 或搜索平台配置。
- GEO 工程提高可抓取性与事实一致性，不承诺第三方模型必然收录、引用或推荐。
